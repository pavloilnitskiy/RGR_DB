# RGR_DB
Проектування бази даних та ознайомлення з базовими операціями СУБД PostgreSQL

Концептуальна модель предметної області “Онлайн-платформа для бронювання місць у громадському транспорті.”
В логічній моделі предметної області “Онлайн-платформа для бронювання квитків” виділяються наступні сутності та зв’язки між ними:

Сутність “Вокзал” (Station) з атрибутами: station_id, назва, місто, тип вокзалу;

Сутність “Маршрут” (Route) з атрибутами: route_id, пункт призначення, відстань;

Сутність “Рейс” (Trip) з атрибутами: trip_id, тип транспорту, час відправлення, ціна;

Сутність “Пасажир” (Passenger) з атрибутами: passenger_id, повне ім’я, телефон;

Сутність “Бонус-карта” (BonusCard) з атрибутами: card_id, рівень картки;

Сутність “Квиток” (Ticket) з атрибутами: ticket_id, номер місця, час покупки.

Опис зв'язків:
Один вокзал може обслуговувати декілька маршрутів, тому між сутностями “Вокзал” та “Маршрут” зв’язок R (1:N).
За одним маршрутом може здійснюватися багато рейсів (у різний час або різним транспортом), тому між сутностями “Маршрут” та “Рейс” зв’язок R (1:N).
Один пасажир може придбати квитки на декілька рейсів, і на один рейс може бути продано квитки багатьом пасажирам. Тому між сутностями “Пасажир” та “Рейс” існує зв’язок R (N:M), який реалізується через проміжну сутність “Квиток”.
У одного пасажира може бути тільки одна активна бонусна картка, тому між сутностями “Пасажир” та “Бонус-карта” зв’язок R (1:1).

Оскільки відношення між пасажирами та рейсами R (N:M), потрібно створити ще одну додаткову таблицю з пасажирами та рейсами під назвою “Ticket”.
Оскільки відношення між пасажирами та бонусними картками R (1:1), потрібно створити ще одну додаткову таблицю з пасажирами та картками під назвою “Passenger_Card”.
